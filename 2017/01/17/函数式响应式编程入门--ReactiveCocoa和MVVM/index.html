函数式响应式编程入门 -- ReactiveCocoa和MVVM

发表于 2017-01-17   |   分类于 ReactiveCocoa   |  

##什么是函数式响应式编程? 什么是ReactiveCocoa？

ReactiveCocoa结合了几种编程风格：

  • 函数式编程（Functional Programming）：使用高阶函数，例如函数用其他函数作为参数。

  • 响应式编程（Reactive Programming）：关注于数据流和变化传播。

    ReactiveCocoa是由 Github 开源的一个应用于 iOS 和 OS X 开发的新框架,ReactiveCocoa被描述为函数响应式编程（FRP）框架。RAC 具有函数式编程和响应式编程的特性。它主要吸取了 .Net 的 Reactive Extensions 的设计和实现。但是，相对于传统的 MVC 架构，ReactiveCocoa 的函数式编程方式的学习曲线陡峭。

##ReativeCocoa入门学习
作为一个才刚刚入门函数式响应式编程的小学生记录一下学习RAC(ReactiveCocoa框架)的一些经验,刚开始学习RAC的时候很难改掉之前的编程习惯。苹果一直推荐大家使用MVC架构思想去编程。但是这样会产生很多东西都堆积在Controller(控制器)中，这样代码高耦合。所以一般情况下RAC会配合着MVVM架构只用会产生不一样的编程效果，本文不介绍MVVM架构思想。如果作为一个iOS开发者一定知道你写的每一行代码几乎都是在响应某个事件，例如按钮的点击，收到网络消息，属性的变化（通过KVO）或者用户位置的变化（通过CoreLocation）。但是这些事件都用不同的方式来处理，比如action、delegate、KVO、callback等。RAC为事件定义了一个标准接口，从而可以使用一些基本工具来更容易的连接、过滤和组合。使用RAC编程主要是所有响应事件转化成信号的思想。可能信号会相对有些抽象，在我初期学习的时候虽然网上的资料很丰富，但是学习曲线很长需要时间去吃透。
